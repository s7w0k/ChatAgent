# 智扫通智能客服系统架构文档

## 一、项目概述

本项目是一个基于 **大语言模型 (LLM)** 的**智能客服系统**，专门为扫地机器人和扫拖一体机器人产品提供售前咨询、售后支持、使用指导等服务。

系统采用现代化的 **Agent 架构**，结合 **RAG (Retrieval-Augmented Generation)** 检索增强生成技术，能够基于专业知识库回答用户问题，并通过 ReAct 推理模式实现复杂任务处理。

### 核心特性

- **智能问答**: 基于 RAG 知识库的精准问答
- **ReAct 推理**: 模拟人类思考过程的智能代理
- **工具调用**: 支持多种业务工具（天气、用户数据、报告生成等）
- **动态提示词**: 根据场景自动切换提示词策略
- **中间件监控**: 全链路执行监控与日志记录
- **Web 交互**: Streamlit 友好的聊天界面

## 三、核心技术详解

### 3.1 LangChain 与 LangGraph

项目使用 **LangChain** 作为 LLM 应用开发框架，核心组件包括：

| 组件 | 说明 |
|------|------|
| `create_agent` | 创建 LangChain Agent，自动处理工具调用循环 |
| `PromptTemplate` | 提示词模板，支持变量替换 |
| `StrOutputParser` | 输出解析器，将 LLM 输出转为字符串 |
| `Tool` / `@tool` | 工具装饰器，定义工具的描述和参数 |

LangChain 的 `create_agent` 基于 **LangGraph** 构建，LangGraph 是一个用于构建有状态、多代理系统的库，支持：
- 工具节点 (Tool Node)
- 条件边 (Conditional Edge)
- 状态管理 (AgentState)
- 中间件机制 (Middleware)

### 3.2 ReAct 推理模式

ReAct (Reasoning + Acting) 是一种结合推理和行动的 LLM 调用模式：

```
思考 (Thought) → 行动 (Action) → 观察 (Observation) → 再思考 ...
```

项目中的 `main_prompt.txt` 明确定义了 Agent 的 ReAct 执行流程：

1. **分析需求**: 判断用户问题是否需要调用工具
2. **选择工具**: 根据需求选择合适的工具
3. **执行工具**: 调用工具获取结果
4. **评估结果**: 判断工具返回是否足以回答问题
5. **生成回答**: 整合信息生成最终回复

### 3.3 RAG (检索增强生成)

RAG 是本系统的核心知识问答技术：

```
用户查询 → 向量检索 → 获取参考资料 → LLM 生成回答
```

**技术要点：**

- **向量化**: 使用阿里云 DashScope 的 `text-embedding-v4` 模型将文本转为 1536 维向量
- **向量存储**: Chroma 向量数据库，支持持久化存储和相似度检索
- **文本分块**: `RecursiveCharacterTextSplitter` 递归分块，chunk_size=200，重叠=20
- **相似度检索**: Top-K 检索，K=3
- **MD5 去重**: 基于文件 MD5 哈希值避免重复加载

### 3.4 MCP 中间件机制

项目实现了 LangChain 的中间件接口，用于：

| 中间件 | 功能 |
|--------|------|
| `@wrap_tool_call` (monitor_tool) | 监控工具调用，记录输入输出日志 |
| `@before_model` (log_before_model) | 模型调用前日志记录 |
| `@dynamic_prompt` (report_prompt_switch) | 动态切换系统提示词 |

**动态提示词切换**是一个重要特性：
- 普通问答场景：使用 `main_prompt.txt`
- 报告生成场景：使用 `report_prompt.txt`

通过 `fill_context_for_report` 工具触发上下文标记，中间件检测到标记后自动切换提示词。

### 3.5 工具系统

项目定义了 7 个业务工具：

| 工具名称 | 功能 | 入参 | 用途 |
|----------|------|------|------|
| `rag_summarize` | RAG 知识检索 | query: str | 获取专业知识库内容 |
| `get_weather` | 天气查询 | city: str | 查询城市天气状况 |
| `get_user_location` | 获取用户城市 | 无 | 获取用户地理位置 |
| `get_user_id` | 获取用户ID | 无 | 获取当前用户标识 |
| `get_current_month` | 获取当前月份 | 无 | 获取系统当前月份 |
| `fetch_external_data` | 获取外部数据 | user_id, month | 获取用户使用记录 |
| `fill_context_for_report` | 报告上下文注入 | 无 | 触发提示词切换 |

## 九、总结

本项目是一个典型的 **LLM Agent 应用** 架构实践，完美展示了：

1. **RAG 技术的工程落地** - 从知识库构建到向量检索的完整链路
2. **ReAct 推理模式** - 模拟人类思考的工具调用流程
3. **中间件设计** - 灵活扩展的请求处理链
4. **动态提示词工程** - 场景感知的提示词切换
5. **模块化架构** - 清晰的分层和职责划分

通过本项目的学习，可以掌握构建生产级 LLM 应用的核心技术，为后续更复杂的 Agent 系统开发奠定坚实基础。
